안녕하세요 심상원 과장님
솔루션개발사업부 조준형 입니다.

헬스키퍼 예약 시스템 오픈을 위해, 시스템 알림 메일 발송 계정 설정을 요청드립니다.

1. 배경
헬스키퍼는 예약 확정, 탈락·재신청 안내, 이메일 인증 등 시스템 알림 메일을 자동 발송합니다.
발신 주소는 noreply@ibank.co.kr (발신 전용, 회신 불필요)를 사용할 예정입니다.

기존에 검토했던 테라웹메일(iwinv) 방식은 ibank.co.kr 도메인 MX가 이미 Microsoft 365로 설정되어 있어
루트 도메인 메일 호스팅과 충돌하며, 테라웹메일은 서브도메인 설정도 지원하지 않아 사용이 어렵습니다.

따라서 직원 메일(M365) 환경을 그대로 유지하면서, 동일 도메인에서 시스템 발송만 하도록 Microsoft 365 SMTP 연동을 요청드립니다.

중요: ibank.co.kr MX 레코드는 변경하지 않습니다. (직원 메일 수신에 영향 없음)

2. 요청 사항
(1) 발송 계정 생성

- 계정 이메일 : 	noreply@ibank.co.kr
- 용도 : 헬스키퍼 시스템 알림 발송 전용 (수신·회신 불필요)
- 권한 : 일반 사용자 (관리자 권한 불필요)
- 라이선스 : Exchange Online 1개

(2) SMTP AUTH 허용
헬스키퍼 서버가 smtp.office365.com:587으로 SMTP 인증 발송을 합니다.

Exchange 관리 센터 → 설정 → 메일 흐름 → 인증된 SMTP 활성화 또는
noreply@ibank.co.kr 사서함에 대해 SMTP 클라이언트 인증 허용
(Set-CASMailbox -Identity noreply@ibank.co.kr -SmtpClientAuthenticationDisabled $false)

보안 정책(조건부 액세스, Security Defaults 등)으로 SMTP가 차단되지 않도록 noreply@ibank.co.kr 계정 예외 허용도 함께 검토 부탁드립니다.

(3) 앱 비밀번호 발급
SMTP 연동용으로 해당 계정에 앱 비밀번호 1개 발급 후 전달 부탁드립니다.

(4) DNS·인증 상태 확인 (변경 최소화)
아래 항목이 정상인지 확인만 부탁드립니다. (이미 M365 사용 중이면 대부분 설정되어 있을 것으로 예상됩니다.)

- SPF : ibank.co.kr TXT에 include:spf.protection.outlook.com 포함
- DKIM : Exchange 관리 센터에서 ibank.co.kr DKIM Enabled 상태
- DMARC : _dmarc.ibank.co.kr 레코드 존재 여부 (없으면 추가 검토)

* MX 레코드는 Microsoft 365 그대로 유지해 주세요.

바쁘신 중에 요청드려 죄송합니다.
추가로 필요한 정보가 있으시면 말씀해 주시면 바로 전달드리겠습니다.

감사합니다.
