# 아이뱅크 안마서비스(헬스키퍼) 페이지별 상세 설계서

> 본 문서는 「아이뱅크 안마서비스(헬스키퍼) 예약 시스템 설계서 초안」을 토대로
> 개별 메뉴/페이지 단위의 상세 기능정의서와 비즈니스 로직 시나리오를 정리한 문서이다.
> 화면 설계(와이어프레임) 단계의 기반 자료로 활용한다.

---

## 0. 공통 정의

### 0.1 용어 정의
| 용어 | 정의 |
|------|------|
| 슬롯(Slot) | 예약 단위 시간 (예: 6/15(월) 13:30~14:00). 하루 4슬롯, 주당 최대 20슬롯 |
| 신청(요청) | 회원이 슬롯을 선택해 접수한 상태. 확정 대기 |
| 확정(완료) | 예약이 최종 확정된 상태 (관리자 확정 또는 재신청 즉시 확정) |
| 탈락 | 우선권 경합에서 밀려 확정되지 못한 신청 |
| 취소 | 마감 전 신청자/관리자가 취소한 신청 |
| 우선권 | 동일 슬롯 중복 신청 시 배분 기준 (마지막 이용일 오래된 순 → 동률 시 신청 시각 빠른 순) |
| 마지막 이용일 | 회원이 마지막으로 안마서비스를 받은(확정 완료 기준) 날짜 |
| 예약 오픈 | 매주 수요일 09:00, 차주 일정 신청 시작 |
| 예약 마감 | 매주 수요일 17:00 정각 (실제 접수 16:59까지) |
| 재신청 창 | 마감 직후(수 17:00) ~ 다음날(목) 17:00. 탈락자 한정 선착순 즉시 확정 |
| 대상 기간 | 신청 가능한 차주 월~금 (5일) |

### 0.2 운영 시간 슬롯
| 타임 | 시간 |
|------|------|
| 1타임 | 13:30 ~ 14:00 |
| 2타임 | 14:30 ~ 15:00 |
| 3타임 | 15:30 ~ 16:00 |
| 4타임 | 16:30 ~ 17:00 |

### 0.3 시간대별 시스템 상태(State)
| 상태 코드 | 기간 | 회원 화면 동작 |
|-----------|------|----------------|
| BEFORE_OPEN | 수요일 09:00 이전 | "예약 오픈 전" 안내, 신청 불가 |
| OPEN | 수요일 09:00 ~ 16:59 | 신청·취소 활성 |
| REAPPLY | 수요일 17:00 ~ 목요일 17:00 | 탈락자 한정 재신청 가능 |
| CLOSED | 목요일 17:00 ~ 다음 수요일 09:00 | "예약 마감", 신청 불가 |

### 0.4 예약 상태(Status)
| 상태 | 설명 | 전환 가능 |
|------|------|-----------|
| 신청(요청) | 일반 신청 접수, 확정 대기 | → 확정 / 탈락 / 취소 |
| 확정(완료) | 최종 확정 (완료 메일 발송) | 재신청 확정은 취소 불가 |
| 탈락 | 우선권 밀림 (재신청 안내 대상) | → 재신청 시 신규 확정 |
| 취소 | 마감 전 취소 | (종료) |

### 0.5 공통 정책
- 모든 시간 기준은 **KST(Asia/Seoul)** 단일 적용.
- 안마사 **1명**, 슬롯당 **1명만 확정** 가능.
- 동시성 발생 지점(재신청 즉시 확정)은 **DB 슬롯 단위 락 / 유니크 제약**으로 처리.
- 메일 발송 실패 시 **재시도 큐**에 적재하고 관리자 대시보드에 발송 상태를 표시.
- 회원의 신청 취소는 **마지막 이용일에 영향 없음**(우선권 불변).

# Part A. 회원(사용자) 영역

## A-1. 홈 / 서비스 소개

### A-1.1 개요
- **경로(예시)**: `/`
- **접근 권한**: 전체(비회원 포함)
- **목적**: 헬스키퍼 서비스 소개 및 진입 동선 제공

### A-1.2 기능 정의
| No | 기능명 | 설명 | 비고 |
|----|--------|------|------|
| A1-01 | 서비스 소개 노출 | 서비스 설명, 운영시간(13:30~17:00, 4타임), 예약 정책 요약 | 정적/CMS |
| A1-02 | 현재 상태 배너 | 현재 시스템 상태(0.3)에 따른 안내 문구 노출 | 상태 분기 |
| A1-03 | CTA 분기 | 비회원: [회원가입]/[로그인], 회원: [예약하기] | 로그인 상태 분기 |

### A-1.3 상태별 배너 문구(예시)
| 상태 | 문구 |
|------|------|

| OPEN | "차주 예약 신청이 가능합니다. (오늘 17:00 마감)" |
| REAPPLY | "탈락자 재신청 기간입니다. (내일 17:00까지)" |
| CLOSED | "현재 예약 접수 기간이 아닙니다. 다음 오픈: 수요일 09:00" |

## A-2. 회원가입

### A-2.1 개요
- **경로(예시)**: `/signup`
- **접근 권한**: 비회원
- **목적**: 이메일 인증 기반 회원가입

### A-2.2 입력 항목
| 항목 | 필수 | 유효성 |
|------|------|--------|
| 이름 | Y | 1~50자 |
| 이메일 | Y | 이메일 형식, 중복 불가 |
| 비밀번호 | Y | 영문+숫자+특수문자, 최소 길이 정책 |
| 비밀번호 확인 | Y | 비밀번호 일치 |

### A-2.3 기능 정의
| No | 기능명 | 설명 | 제약 |
|----|--------|------|------|
| A2-01 | 회원정보 입력/검증 | 필수값·형식 검증 | 클라이언트+서버 이중 검증 |
| A2-02 | 이메일 중복 검사 | 기존 회원 이메일 중복 여부 확인 | 중복 시 가입 불가 |
| A2-03 | 인증 메일 발송 | 가입 즉시 인증 링크 발송 | 토큰 만료 24h |
| A2-04 | 가입 상태 저장 | 인증 전까지 `미인증` 저장 | 미인증 시 로그인 제한 |

### A-2.4 비즈니스 로직 시나리오
**[정상 가입]**
1. 회원이 정보 입력 후 [가입] 클릭
2. 시스템: 이메일 형식·중복 검사 → 비밀번호 규칙 검사
3. 통과 시 회원 상태 `미인증`으로 저장
4. 인증 토큰 생성(24h) → 인증 메일 발송
5. "인증 메일을 발송했습니다. 메일을 확인해주세요." 안내

**[중복 이메일]**
1. 입력 이메일이 기존 회원과 동일
2. "이미 가입된 이메일입니다." 알럿 → 가입 중단

**[비밀번호 불일치]**
1. 비밀번호/확인 불일치
2. "비밀번호가 일치하지 않습니다." 안내 → 제출 차단

## A-5. 예약 캘린더 (예약하기 · 핵심 페이지)

### A-5.1 개요
- **경로(예시)**: `/reservation`
- **접근 권한**: 회원(인증완료)
- **목적**: 차주(월~금) 슬롯 조회 및 예약 신청/취소
- **핵심 의존**: 현재 시스템 상태(0.3), 안마사 휴가, 슬롯 신청/확정 현황

### A-5.2 화면 구성 요소
| 영역 | 구성 | 설명 |
|------|------|------|
| 상태 배너 | 현재 상태 안내 | OPEN/REAPPLY/CLOSED 등 분기 |
| 주간 캘린더 | 차주 월~금 × 4타임 그리드 | 슬롯 상태별 표시 |
| 슬롯 셀 | 슬롯별 [신청]/[취소] 버튼 | 상태별 동작 분기 |
| 정책 안내 | 정책 요약 | "신청 ≠ 확정" 명확 고지 |

### A-5.3 슬롯 표시 정의
| 표시 | 조건 | 동작 |
|------|------|------|
| 신청 가능 | OPEN & 비휴가 & 내 미신청 | [신청] 활성 |
| 내 신청됨 | 내가 신청한 슬롯 | [취소] 활성(마감 전) |
| 타인 신청 포함 | 동일 슬롯 타 회원 신청 존재 | 중복 신청 가능(우선권 경합 안내) |
| 휴가/예약불가 | 안마사 휴가 날짜 | 비활성 |
| 마감/오픈전 | BEFORE_OPEN/CLOSED | 전체 비활성 |

> 동일 슬롯 중복 신청이 허용되며, 확정은 마감 시점(우선권 정렬) 또는 관리자 처리에서 결정된다.

### A-5.4 기능 정의
| No | 기능명 | 설명 | 제약 |
|----|--------|------|------|
| A5-01 | 차주 일정 조회 | 차주 월~금 슬롯·휴가·상태 조회 | OPEN/REAPPLY일 때 신청 가능 |
| A5-02 | 슬롯 신청 | 슬롯 선택 후 신청 접수 | OPEN 시간 내, 비휴가 슬롯만 |
| A5-03 | 중복 신청 안내 | 동일 슬롯 타인 신청 존재 시 안내 | "우선권에 따라 확정" |
| A5-04 | 신청 접수 알림 | "신청 접수 완료(확정 아님)" 안내 | 확정과 구분 |
| A5-05 | 신청 취소 | 마감(17:00) 전까지 취소 | 마감 후 비활성 |
| A5-06 | 재신청 진입 | REAPPLY & 탈락자 한정 재신청 화면 진입 | 비대상자 차단 |

### A-5.5 비즈니스 로직 시나리오

**[시나리오 1: 정상 신청 (OPEN, 단독 신청)]**
1. 회원이 수요일 09:00~16:59 사이 예약 캘린더 진입
2. 시스템 상태 = OPEN 확인 → 차주 슬롯 노출(휴가는 비활성)
3. 비어있는 슬롯 선택 → [신청]
4. 시스템: OPEN 여부·휴가 여부·동일 회원 동일 슬롯 중복 여부 검증
5. 통과 시 `신청(요청)` 상태로 접수
6. "신청이 접수되었습니다. (관리자 확정 후 예약이 완료됩니다.)" 안내

**[시나리오 2: 동일 슬롯 중복 신청]**
1. 회원 B가 이미 회원 A가 신청한 슬롯을 선택
2. 시스템: 중복 신청 허용 안내 노출
   - "해당 시간은 다른 신청자가 있습니다. 우선권(마지막 이용일)에 따라 마감 시점에 확정됩니다."
3. 회원 B [신청] → `신청(요청)` 접수
4. 마감 시점에 우선권 정렬로 확정/탈락 결정 (B-7 마감 배치 참조)

**[시나리오 3: 마감 전 취소]**
1. 회원이 마이페이지 또는 캘린더에서 본인 신청 슬롯 [취소]
2. 시스템: 현재 OPEN(마감 전) 여부 확인
3. 마감 전이면 상태 `취소` 처리 → 슬롯 신청 해제
4. "신청이 취소되었습니다." (※ 마지막 이용일 불변, 우선권 영향 없음)

**[시나리오 4: 마감 후 취소 시도]**
1. 회원이 마감(17:00) 이후 취소 시도
2. 시스템: CLOSED/REAPPLY 상태 확인
3. "마감 이후에는 취소할 수 없습니다." 알럿 → 취소 거절

**[시나리오 5: 오픈 전 / 마감 후 신청 시도]**
1. 회원이 BEFORE_OPEN 또는 CLOSED 시간에 신청 시도
2. 시스템: 신청 불가 상태 확인
3. 신청 버튼 비활성 + "현재 예약 신청 기간이 아닙니다." 안내

**[시나리오 6: 휴가 슬롯 신청 시도]**
1. 회원이 휴가로 지정된 날짜 슬롯 선택 시도
2. 시스템: 해당 날짜 휴가 여부 확인
3. 비활성 표시 + "안마사 휴무일입니다." 안내 → 신청 차단

## A-6. 탈락자 재신청 (선착순 · 즉시 확정 · 취소 불가)

### A-6.1 개요
- **경로(예시)**: `/reservation/reapply`
- **접근 권한**: REAPPLY 상태 + 해당 주차 탈락자
- **목적**: 마감 후 비어있는 슬롯을 선착순으로 즉시 확정
- **운영 시간**: 마감 직후(수 17:00) ~ 다음날(목) 17:00

## B-7. 마감 시점 자동 처리 (배치)

### B-7.1 개요

- **목적**: 일반 신청 마감, 우선권 정렬, 탈락 처리, 탈락 안내, 재신청 창 오픈

|----|--------|------|

| B7-02 | 우선권 정렬 | 동일 슬롯 중복 신청 건 우선권 정렬 |
| B7-03 | 탈락 처리 | 우선권 밀린 신청자 `탈락` 처리 |
| B7-04 | 재신청 안내 메일 | 탈락자에게 빈 슬롯·재신청 안내 자동 발송 |

1. 수요일 17:00 도달 → 신청/취소 차단(상태 변경)
2. 슬롯별 신청 그룹 집계
3. 단독 신청 슬롯: 확정 대상으로 분류 (관리자 확정 또는 정책에 따른 처리)

6. 상태 REAPPLY 전환 → 탈락자 한정 재신청 창 오픈
7. 목요일 17:00 → 상태 CLOSED 전환, 모든 신청 종료

> 본 설계서에서는 마감 시 우선권 정렬·탈락 처리를 자동 수행하고, 일반 신청 슬롯의 최종 확정은 관리자 확정(B-5)을 거치는 구조로 본다. (정책에 따라 마감 시 자동 확정으로 전환 가능 — 운영 설정에서 조정)

1. 관리자가 오픈(수 09:00) 전에 휴가 관리 진입
2. 캘린더에서 차주 휴가 날짜 지정 → 저장
3. 오픈 시 해당 날짜 슬롯 회원 화면에 예약 불가(비활성) 반영

**[오픈 후 등록 시도]**
1. 관리자가 오픈 이후 휴가 등록 시도
2. "예약 오픈 후에는 해당 주차 휴가를 등록할 수 없습니다." → 차단

# Part D. 예약 라이프사이클 (통합 흐름)

## D-1. 전체 흐름도

[오픈 전] 관리자 차주 휴가 미리 등록
   │
   ▼
[수 09:00] 차주 일정 오픈 (이후 휴가 등록 불가)
   │
   ▼
[수 09:00~16:59] 회원 신청 (마감 전 취소 가능 / 취소는 우선권 영향 없음)
   │                              ┌─ 동일 슬롯 중복 신청
   ▼                              ▼
(신청/요청)                  [우선권 정렬]
   │                         1순위: 마지막 이용일 오래된 순
   │                         2순위: 동률 시 신청 시각 빠른 순
   │                         (이력 없음: 관리자 수동 판단)
   │                              │
   │                    ┌─────────┴─────────┐
   │             [우선권 보유]          [우선권 탈락]
   │             관리자 확정             탈락 처리
   │                  │                     │
   │             (확정/완료)          탈락/재신청 안내 메일
   │             완료 메일 발송              │
   │             마지막 이용일 갱신          │
   ▼                                        ▼
[수 17:00 마감] (이후 취소 불가)   [재신청 창 오픈: 수17:00~목17:00]
                                            │
                                  탈락자, 빈 슬롯 선착순 재신청
                                  (이미 확정 시 "이미 예약 완료" 거절)
                                            │
                                  즉시 확정(취소 불가) → 완료 메일
                                            │
                                            ▼
                          [목 17:00 재신청 창 마감] → 모든 신청 종료
                                            │
                          다음 차주 일정 = 다음 주 수 09:00 신규 오픈

## D-2. 단계별 상태 전이표

| 단계 | 시점 | 트리거 | 시스템 상태 | 예약 상태 변화 | 주요 처리 |
|------|------|--------|-------------|----------------|-----------|
| 0. 휴가 등록 | 오픈 전 | 관리자 | BEFORE_OPEN | - | 차주 휴가 등록(오픈 후 불가) |
| 1. 오픈 | 수 09:00 | 스케줄러 | BEFORE_OPEN → OPEN | - | 차주 슬롯 신청 가능, 휴가 등록 차단 |
| 2. 신청 | 수 09:00~16:59 | 회원 | OPEN | (없음) → 신청(요청) | 슬롯 신청 접수, 중복 신청 허용 |
| 3. 취소 | 수 09:00~16:59 | 회원 | OPEN | 신청(요청) → 취소 | 슬롯 해제, 우선권 영향 없음 |
| 4. 마감 | 수 17:00 | 스케줄러 | OPEN → REAPPLY | - | 신청/취소 차단 |
| 5. 우선권 정렬 | 수 17:00 | 배치 | REAPPLY | 신청(요청) 분류 | 슬롯별 우선권 정렬 |
| 6-1. 확정 | 수 17:00 이후 | 관리자/배치 | REAPPLY | 신청(요청) → 확정 | 완료 메일, 마지막 이용일 갱신 |
| 6-2. 탈락 | 수 17:00 | 배치 | REAPPLY | 신청(요청) → 탈락 | 탈락/재신청 안내 메일 |
| 7. 재신청 | 수17:00~목17:00 | 탈락자 | REAPPLY | (탈락자) → 확정 | 선착순 즉시 확정, 취소 불가 |
| 8. 재신청 마감 | 목 17:00 | 스케줄러 | REAPPLY → CLOSED | - | 모든 신청 종료 |
| 9. 대기 | 목17:00~다음 수09:00 | - | CLOSED | - | 다음 사이클 대기 |

## D-3. 핵심 분기 로직 시나리오

**[분기 1: 단독 신청 슬롯]**
1. 마감 시점 기준 해당 슬롯에 신청자 1명
2. 우선권 경합 없음 → 확정 대상으로 분류
3. 관리자 확정 처리 → 확정/완료, 완료 메일 발송, 마지막 이용일 갱신

**[분기 2: 중복 신청 슬롯 (이력 보유)]**
1. 마감 시점 기준 동일 슬롯 다수 신청
2. 우선권 정렬: 마지막 이용일 오래된 순 → 동률 시 신청 시각 빠른 순
3. 1순위 확정, 나머지 탈락 → 탈락자에게 재신청 안내 메일

**[분기 3: 중복 신청 슬롯 (전원 이력 없음)]**
1. 신청자 전원 "이력 없음"으로 자동 우선권 판단 불가
2. 관리자가 수동으로 1명 확정, 나머지 탈락 처리

**[분기 4: 탈락자 재신청]**
1. 탈락자가 재신청 창(수17:00~목17:00) 내 빈 슬롯 선택
2. 선착순 즉시 확정 → 완료 메일 발송(취소 불가)
3. 동시 충돌 시 먼저 도달한 신청 확정, 후순위는 "이미 예약이 완료된 날짜 및 시간대입니다." 거절

**[분기 5: 사이클 종료]**
1. 목 17:00 재신청 창 마감 → CLOSED 전환
2. 모든 신청/재신청 차단
3. 다음 차주 일정은 다음 주 수요일 09:00에 신규 오픈

## D-4. 상태 일관성 규칙

- `확정(완료)` 상태에서만 회원의 **마지막 이용일**이 갱신된다. (`취소`/`탈락`은 미반영)
- 재신청으로 인한 `확정`은 취소가 불가능하므로 상태 역전(확정 → 취소)이 발생하지 않는다.
- 마감(수 17:00) 이후에는 어떤 경로로도 `신청(요청) → 취소` 전이가 불가능하다.
- 한 슬롯은 동시에 둘 이상의 `확정`을 가질 수 없다. (슬롯 단위 유니크 제약)

# Part F. 정책 확정 사항 (반영 완료)

1. **재신청 취소**: 즉시 확정 건이므로 취소 불가
2. **노쇼 처리**: 확정 후 노쇼 별도 처리 없음
3. **휴가 등록 시점**: 예약 오픈 전에만 가능, 오픈 후 등록 불가
4. **재신청 동시 클릭**: 이미 확정 슬롯이면 "이미 예약이 완료된 날짜 및 시간대입니다." 알럿 후 거절
5. **첫 운영 시 이력 없는 사용자**: "이력 없음" 표시, 관리자가 수동 확정
6. **회원 취소의 우선권 영향**: 영향 없음 (마감 전 취소는 마지막 이용일 미반영)
